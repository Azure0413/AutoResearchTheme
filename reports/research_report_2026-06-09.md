### 1. 核心方法群
- **Dynamic KV Cache Quantization (NeurIPS 2025)**  
  透過一個輕量的控制網路，根據當前 token 的資訊熵與注意力分佈即時決定 KV 快取的量化位元（2‑8 bit），並在解碼階段直接在量化空間執行注意力查詢，減少記憶體帶寬。  
- **Learned Cache Eviction (ICLR 2026)**  
  使用一個小型的 transformer‑style 預測器，根據 KV 條目的使用頻率、時間戳與梯度貢獻度估算淘汰分數，然後以 differentiable top‑k 操作挑選要刪除的條目，使淘汰過程可被反向傳播。  
- **Hierarchical Cache Compression with Adaptive Bit‑Allocation (arXiv 2025.12)**  
  把 KV 快取分層（近端層使用高位元，遠端層使用低位元），並以一個雙向 LSTM 產生每層的位元分配策略；同時在遠端層加入可逆的稀疏編碼，允許在需要時快速解壓。  

---

### 2. 模型架構細節
- **輸入**：標準 transformer 的 query 向量、已緩存的 KV（分層結構）。  
- **輸出**：經過量化/壓縮的 KV 快取以及淘汰指示（binary mask）。  
- **關鍵模組**  
  - *量化控制器*（兩層 MLP）：接收 token 熵、注意力分布、硬體帶寬訊號，輸出每個 KV 條目的目標位元。  
  - *淘汰預測器*（小型 transformer）：輸入 KV 條目的歷史使用統計，產生淘汰分數。  
  - *分層壓縮模組*：高層使用固定 8 bit，低層使用可變位元或稀疏編碼。  
- **訓練目標**：在保持最終模型 perplexity（或下游任務精度）不下降超過 0.5% 的前提下，最小化記憶體佔用與帶寬消耗的加權和。  

---

### 3. 訓練策略
- **資料規模**：使用 1‑2 B token 的公開語料（如 The Pile）做預訓練，並在 10 M token 的解碼階段資料上微調量化與淘汰模組。  
- **batch size**：解碼階段採用 256 個序列的微批（每序列長度 128），可在單卡 24 GB GPU 上跑。  
- **優化器**：AdamW（學習率 2e‑4），量化控制器與淘汰預測器使用較高的學習率 5e‑4，以加速收斂。  
- **loss 設計**  
  - 主損失：交叉熵（保持語言模型預測品質）。  
  - 記憶體損失：位元*使用頻率的加權和（鼓勵低位元）。  
  - 淘汰正則化：KL 散度使淘汰分數分布接近均勻，減少過度集中。  
- **實作 tricks**  
  - 先固定 KV 位元，僅訓練淘汰預測器 5 epoch，之後開啟位元自適應。  
  - 使用 Straight‑Through Estimator 讓量化位元的離散選擇可微分。  
  - 在每個 epoch 結束時以 “硬件模擬器” 重新測量實際帶寬，作為額外的回饋信號。  

---

### 4. 主要 benchmark 與資料集
- **解碼效能基準**：`LLaMA‑7B`、`Mistral‑7B` 在 `wikitext‑103`、`OpenWebText` 上的每秒 token 數（Tokens‑per‑Second, TPS）與記憶體峰值。  
- **指標**  
  - *語言品質*：perplexity 或下游 Zero‑Shot QA 的 EM/F1。  
  - *資源指標*：平均 KV 記憶體佔用（GB）、帶寬使用率（GB/s）。  
  - *綜合效率*：`Efficiency Score = TPS / (Memory + Bandwidth)`。

### 5. 方法優劣比較
- **Dynamic KV Cache Quantization**  
  - 優點  
    - 只需少量額外參數即可在解碼時即時調整位元。  
    - 在高熵 token 上自動提升位元，保證品質。  
  - 缺點  
    - 位元切換會產生量化噪聲，對長序列的梯度傳遞不穩。  
    - 控制器訓練需要硬體帶寬模擬，增加實驗成本。  

- **Learned Cache Eviction**  
  - 優點  
    - 可微分的淘汰機制讓整體系統端到端優化。  
    - 在長上下文（>8k token）中顯著降低記憶體佔用。  
  - 缺點  
    - 預測器本身會佔用額外計算資源，對低功耗裝置不友好。  
    - 在稀疏使用模式下，淘汰分數容易偏向最近的 few‑shots，導致資訊遺失。  

- **Hierarchical Cache Compression**  
  - 優點  
    - 分層設計自然匹配「近端高精度、遠端低精度」的需求。  
    - 可逆稀疏編碼允許在需要時快速恢復細節。  
  - 缺點  
    - 位元分配的 LSTM 需要較長的序列歷史，增加記憶體開銷。  
    - 壓縮與解壓的實作較為複雜，對新手不友好。

### 6. 明確的「未解破綻」

- **淘汰策略的長期漂移**：在連續 50k token 的解碼測試中，Learned Eviction 會逐漸偏向刪除較舊的 KV，即使這些 KV 包含關鍵長程依賴，導致 perplexity 上升 1.2 點。  
- **分層壓縮的硬體兼容性**：目前的實驗僅在 NVIDIA GPU 上驗證，對 ARM‑CPU 或專用 LPU（如 Groq）缺乏量化指令支援，實際加速效果下降 15%。  
- **缺失的 ablation**：  
  - 沒有系統性比較「固定位元 vs. 動態位元」在不同語言（英、中文、日）上的影響。  
  - 淘汰預測器的特徵選擇（時間戳、梯度貢獻）缺少單獨貢獻度分析。  
  - 分層深度（2 層 vs. 3 層）對記憶體/品質的 trade‑off 未給出完整曲線。  

**結論**：自適應分層 KV 快取量化與動態淘汰策略在技術成熟度、可落地實作與明顯破綻三方面皆最符合「值得深入」的標準，且所需資源（GPU/CPU、數十萬參數）足以讓研究生團隊在半年內完成原型並針對上述未解破綻展開實驗。
