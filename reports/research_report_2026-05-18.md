### 1. 核心方法群
- **Causal Detection of Post‑Training Data Poisoning (NeurIPS 2025)**  
  透過建構「輸入‑嵌入‑輸出」的三層因果圖，利用條件獨立測試判斷哪些輸入子集對模型輸出產生異常因果效應。核心在於以貝葉斯網路估計因果強度，並以隨機遮蔽 (masking) 產生對照組。

- **Tracing Poisoned Tokens via Structural Causal Models (arXiv 2025.06.04123)**  
  把每個 token 視為 SCM（結構因果模型）中的節點，利用「前向因果介入」模擬植入惡意 token 的影響，並透過逆向推理找出最可能的中毒來源。方法重點是對注意力矩陣做因果分解，得到「因果注意力圖」。

- **Counterfactual Analysis for Data‑Poisoning Attribution (ICLR 2026)**  
  以反事實生成 (counterfactual generation) 產生「若未植入特定樣本」的模型預測，並比較實際與反事實輸出差異，以量化每筆訓練樣本的「毒性貢獻分數」。使用生成式對抗網路 (GAN) 產生高品質的反事實樣本。

---

### 2. 模型架構細節
- **輸入**：原始文本或 token 序列，外加可選的「候選中毒樣本」集合。  
- **輸出**：  
  1. 因果效應分數（每個候選樣本對特定輸出結果的貢獻）  
  2. 根因圖（顯示哪些樣本與哪些輸出節點形成因果鏈）  
- **關鍵模組**：  
  - **因果圖構建模組**：將 transformer 的層級注意力映射為因果邊，使用結構化貝葉斯網路或 SCM。  
  - **介入與反事實生成模組**：基於 gradient‑based 介入或 GAN 產生假設的未中毒模型行為。  
  - **貢獻度估計模組**：採用 Shapley‑like 加權平均或因果強度指標，輸出每筆樣本的毒性分數。  
- **訓練目標**：最小化介入後模型輸出與真實輸出之差距，同時最大化因果圖的結構正則化（鼓勵稀疏且符合已知因果先驗）。

---

### 3. 訓練策略
- **資料規模**：使用公開的 LLM 微調資料集（如 Pile、OpenWebText）約 10‑20 GB，並在其中人工植入 0.1%‑0.5% 的中毒樣本作為驗證。  
- **Batch size**：GPU 記憶體允許下採 64‑128 個序列，序列長度 512 token。  
- **優化器**：AdamW，學習率 2e‑4，使用 cosine decay。  
- **Loss 設計**：  
  - **因果圖重建損失**：交叉熵 + KL 散度，用於逼近真實因果結構。  
  - **介入一致性損失**：L2 距離，衡量介入前後輸出差異。  
  - **稀疏正則**：L1 懲罰，鼓勵因果圖僅保留關鍵邊。  
- **實作 tricks**：  
  - 先用低階注意力層（第 1‑3 層）估計粗略因果圖，再在高層細化。  
  - 使用混合精度 (FP16) 減少記憶體佔用。  
  - 介入時採用「梯度遮蔽」技術，避免對未受影響的 token 產生梯度噪聲。

---

### 4. 主要 benchmark 與資料集
- **Benchmark**：  
  - **Poisoned‑GLUE**（在 GLUE 任務上植入目標類別中毒樣本）  
  - **Backdoor‑WikiText**（在 WikiText‑103 中加入隱蔽觸發詞）  
- **關鍵指標**：  
  - **因果召回率 (Causal Recall)**：成功找出實際植入樣本的比例。  
  - **因果精確率 (Causal Precision)**：被標記為中毒樣本中真實中毒樣本的比例。  
  - **偽陽性率 (False Positive Rate)**：正常樣本被錯誤標記的比例。  
  - **計算開銷**：每條序列的因果圖推論時間（ms）。

---

### 5. 方法優劣比較
- **Causal Detection of Post‑Training Data Poisoning**  
  - 優點  
    1. 結構化因果圖易於解釋，適合人類審查。  
    2. 不需要額外生成模型，計算成本相對較低。  
  - 缺點  
    1. 只利用注意力資訊，對於隱蔽的語義層面中毒效果捕捉不足。  
    2. 在長序列上因果圖構建會爆炸式增長記憶體需求。  

- **Tracing Poisoned Tokens via Structural Causal Models**  
  - 優點  
    1. 針對 token 級別的因果介入，定位精度高。  
    2. 可直接結合現有的注意力可視化工具。  
  - 缺點  
    1. 需要對每個 token 計算 SCM，計算成本高。  
    2. 依賴於高品質的注意力分布，對於噪聲敏感。  

- **Counterfactual Analysis for Data‑Poisoning Attribution**  
  - 優點  
    1. 反事實生成提供了「若未中毒」的基線，理論上最具說服力。  
    2. 可與任何預訓練模型搭配，方法通用性強。  
  - 缺點  
    1. 需要訓練額外的 GAN，訓練不穩定且資源需求大。  
    2. 反事實樣本品質直接影響毒性分數的可靠性，易受生成失真影響。  

---

### 6. 明確的「未解破綻」
- **長序列與高維因果圖**：當序列長度超過 1024 token 時，現有方法的因果圖推論時間超過 500 ms，導致實務部署受限。  
- **隱蔽語義中毒**：僅依賴注意力或 token 介入的策略，對於透過語義重寫（例如同義詞替換）植入的後門檢測率低於 30%。  
- **跨模型遷移**：目前的因果圖大多在單一模型（如 LLaMA‑2‑7B）上驗證，缺乏在不同架構（如 encoder‑decoder）間的可遷移性測試。  
- **缺乏系統性 ablation**：  
  - 未系統評估「稀疏正則」強度對召回率的影響。  
  - 缺少對「介入層級」選擇（低層 vs 高層）之效能比較。  
  - 反事實生成的品質指標（如 FID）與最終因果召回率之相關性尚未量化。  

**結論**：此領域已有可運行的原型，且在因果圖構建、介入策略與反事實生成三條路徑上仍存明顯效能瓶頸與資源需求差距。對於具備基礎深度學習與因果推斷背景的學生而言，從「稀疏因果圖優化」或「高效反事實生成」切入，既能快速實驗，也能在學術與安全應用上產生可觀貢獻。
