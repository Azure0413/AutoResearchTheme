### 1. 核心方法群
- **動態低秩分解 (Dynamic Low‑Rank KV Cache)**  
  透過在每一步解碼時即時估算 KV 矩陣的奇異值衰減，選擇最小能保留 90% 能量的秩；若序列長度增長或語義變化劇烈，會自動提升秩以避免資訊遺失。  
- **自適應量化 (Adaptive KV Quantization)**  
  依據當前 token 的置信度與注意力分佈，切換 8‑bit、4‑bit或 2‑bit量化策略；置信度低時使用較高位寬以減少量化噪聲，置信度高時則壓縮至更低位寬提升吞吐。  
- **增強式快取修剪 (Reinforcement‑Learning Cache Pruning)**  
  使用小型策略網路根據「快取命中率」與「推理延遲」獎勵，決定哪些 KV 條目可被刪除或合併；策略在離線模擬環境中預訓練，實際推理時僅做前向查詢，開銷極低。

### 2. 模型架構細節
- **輸入**：Transformer 解碼器的前一層隱藏狀態、當前 token 的嵌入、以及已緩存的 KV（Key、Value）矩陣。  
- **輸出**：更新後的 KV 快取（可能已低秩分解或量化）以及當前層的注意力輸出。  
- **關鍵模組**：  
  - **秩估算模組**：輕量的全連接層，根據前一層注意力分佈輸出建議秩。  
  - **位寬選擇器**：根據置信度分數（softmax 溫度）映射至離散位寬集合。  
  - **RL 修剪器**：小型 LSTM/Transformer，輸入快取使用統計（命中率、存活時間），輸出刪除/合併決策。  
- **訓練目標**：最小化「推理延遲 + 壓縮懲罰」的加權和，同時保持最終生成品質（BLEU / Rouge）不下降超過 1%。

### 3. 訓練策略
- **資料規模**：使用 200‑300 億 token 的混合語料（OpenWebText、RedPajama、中文 CC‑100），其中 10% 作為「長序列」子集（> 4k token）以測試動態調整。  
- **Batch size**：每張 GPU 8‑16 個序列，總體累積 batch 約 256，足以穩定估算秩與位寬分佈。  
- **優化器**：AdamW（β1=0.9、β2=0.98），學習率 2e‑4，採線性 warm‑up 前 2k 步，之後 cosine 衰減。  
- **Loss 設計**：  
  - 主體交叉熵損失（保持語言模型品質）。  
  - 壓縮正則化：`L_compress = λ1 * (實際 KV 位元數 / 原始位元數) + λ2 * (秩比例 - 目標秩比例)^2`。  
  - 延遲懲罰：使用測量的 GPU 推理時間乘以係數 λ3 加入總 loss。  
- **實作 tricks**：

### 5. 方法優劣比較
#### 1. `Dynamic Low‑Rank KV Cache for Efficient LLM Inference`
- **優點**  
  - 能根據序列長度自動調整秩，對長文本保持較高注意力保真度。  
  - 計算開銷低，僅需一次 SVD 近似即可。  
- **缺點**  
  - 秩估算依賴全局奇異值分佈，對突發語義變化反應較慢。  
  - 在極短序列（< 256 token）時，低秩壓縮反而增加延遲。  

#### 2. `Adaptive KV Quantization for Transformer Decoding`
- **優點**  
  - 位寬切換簡單，與現有硬體量化指令兼容。  
  - 在高置信度區段可顯著降低記憶體帶寬。  
- **缺點**  
  - 量化噪聲在低置信度區段仍會累積，導致後續 token 錯誤放大。  
  - 需要額外的置信度估算模組，增加參數量。  

#### 3. `Reinforcement‑Learning Cache Pruning with RL`
- **優點**  
  - 能在不影響命中率的前提下自動刪除冗餘 KV，提升記憶體利用率。  
  - RL 策略可根據不同硬體資源自行調整 aggressiveness。  
- **缺點**  
  - 訓練成本高，需大量模擬軌跡才能收斂。  
  - 在極端長序列上，策略可能過度刪除關鍵 KV，導致生成崩潰。  

---

### 6. 明確的「未解破綻」
- **秩/位寬即時調整的延遲**：目前的秩估算與位寬選擇在每個 token 前都需要一次前向 pass，對 1‑2ms/ token 的實時應用仍是瓶頸。  
- **跨語言/跨領域泛化**：在多語言混合或專業領域（醫學、法律）語料上，低秩分解的能量保留率下降 5‑10%，導致生成錯誤率上升。  
- **置信度估算的準確性**：現有的 softmax 溫度作為置信度指標在長序列上呈現「過度自信」現象，量化位寬選擇因此失效。  
- **RL 修剪的穩定性**：缺少對「長期依賴」的顯式獎勵，策略在早期階段傾向刪除遠端 KV，對需要遠距離回溯的任務（如長篇摘要）表現急劇下降。  
- **缺失的 ablation**：  
  - 尚未有系統性比較「固定秩+自適應量化」 vs 「自適應秩+固定量化」的交互效應。  
  - 未針對不同硬體（GPU vs TPU）測試 SVD 計算成本與量化指令延遲的差異。  
  - 缺少對「多模態 KV」（如視覺‑語言模型）壓縮策略的驗證。  

**結論**：動態低秩 KV 快取結合自適應量化與增強式修剪，已具備可實作的基礎，但在即時調整機制、跨領域穩定性與 RL 獎勵設計上仍有明顯空間。學生團隊可先聚焦於「快速秩/位寬預測器」與「置信度校正」兩個子問題，逐步構建完整的動態壓縮框架。
